Amplitude Web Implementation

GTM Browser SDK with instance Name

It is unlikely that you have a hard-coded Amplitude implementation in your website alongside a second one done with Google Tag Manager. Also unlikely that you need a more advanced implementation where you have to ensure the SDK is really initiated before anything else, to have a tight control of order in the basic events ($identify and page view) plus sending to the dataLayer information about Amplitude session, device and user IDs to make it available to other tracking tools or services you may have in your website (so you can merge data from both tools using those IDs as keys) or implement cross-domain tracking BUT if that's an scenario you are experiencing, here you have a way to solve it all in once. Check the code of this page to see it in action.

It uses an Instance Name the GTM analytics browser wrapper so you will see the use of amplitudeGTM._iq[instanceName] to isolate one implementation from the other and avoid polution.

The code, as indicated in the HTML comments, can be added into GTM HTML container with the trigger that works in your scenario and the GTM variables to replace the hard coded ones there.
